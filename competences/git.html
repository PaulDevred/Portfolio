Ma Vision de Git

Git est un outil de versionnage qui est aussi la base de toute collaboration efficace et de toute démarche de développement professionnelle. Il me permet de suivre précisément l'évolution d'un projet, d'expérimenter sans risque, et de revenir en arrière si nécessaire.

Dans un contexte professionnel, il assure la cohérence du code entre plusieurs développeurs et garantit la traçabilité des modifications. Avec l'expérience, je comprends qu'un bon usage de Git repose autant sur la technique que sur la rigueur collective : messages de commit clairs, revues systématiques, et branches bien nommées.

Comment j'utilise Git

1. Dans mes projets personnels

Dès mes premiers projets, j'utilise systématiquement Git pour structurer mon travail. Lors du développement de mes projets, notamment Darwin Simulator, je configure un dépôt Git, définis une convention de branches (main/dev/features), et mets en place des pull requests pour assurer la relecture de code.

Cela permet de collaborer en limitant les erreurs et garantit une meilleure qualité de livraison.

Pratiques mises en place

  • Convention de branches structurée
  • Pull requests systématiques
  • Messages de commit explicites
  • Revue de code entre collaborateurs

→ Voir Darwin Simulator

2. Collaboration en entreprise (IoT Platform)

Chez IoThink-Solutions, Git est au cœur du processus de développement. Je participe à la création et à la maintenance de branches liées à différents projets.

Avec des revues de code, des merges, rebases (parfois interactifs), cherry-pick et revert réguliers, l'utilisation rigoureuse de DevOps permet de conserver nos repositories et leurs historiques cohérents.
